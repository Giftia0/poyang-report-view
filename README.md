# Poyang Report View

鄱阳湖线索/举报系统移动端前端，基于 uni-app 和 Vue 3 开发，提供公众上报、线索查询、办理进度、处理工作台和数据统计等页面。项目配合 `poyang-report-backend` 使用，覆盖从用户登录、实名认证、线索提交到办理反馈的完整移动端流程。

## 项目定位

本项目是面向用户和处理人员的移动端入口：

- 普通用户可以提交举报、咨询、反馈，查看自己的上报记录和办理进度。
- 处理人员可以在工作台查看待办线索，录入处理状态，上传办理材料。
- 数据页面可以查看线索数量、类型占比、办理状态和时间趋势。

项目使用 uni-app 组织页面，适合运行到 H5、App 或小程序端。

## 技术栈

- uni-app
- Vue 3 Composition API
- Pinia
- ECharts
- Moment.js
- 高德地图 JS API / 静态地图
- uni-ui 组件

## 核心功能

### 登录与用户中心

- 手机号验证码登录。
- Token 本地存储和请求头自动携带。
- Pinia 管理登录用户状态。
- 个人中心展示头像、用户名和入口菜单。
- 支持退出登录。

### 实名认证

- 身份证人像面、国徽面上传。
- 上传进度展示。
- 自动读取认证结果并回填姓名、性别、地址、证件号、有效期、签发机关等信息。
- 查询认证状态和认证详情。
- 支持重新提交认证材料。

### 线索上报

- 首页 TabBar 提供“举报”入口，并可选择举报、咨询、反馈三类内容。
- 表单包含标题、详细描述、类型选择、图片材料和位置信息。
- 举报、咨询类内容支持地图选点，记录经纬度、地址、POI、城市、区县、乡镇。
- 图片材料最多支持 9 张，提交后按索引上传。
- 上报完成后跳转到提交结果对应页面。

### 地图能力

- `MapPicker` 组件用于移动选点，基于高德 PositionPicker 获取位置结果。
- `MapContainer` 组件用于区域数据展示，包含地图初始化、行政区边界、点位聚合和自定义聚合样式。
- 列表和详情页通过高德静态地图展示线索位置。

### 线索列表

- “线索数据”页展示近一个月统计指标，并可进入列表查看。
- “我的上报”支持按类别、状态、时间范围筛选。
- “工作台”支持处理人员查看待办线索。
- “线索数据”列表支持全部线索查看，适合管理视角。
- 列表项展示标题、描述、类型、地区、状态和时间。

### 详情与状态流转

- 举报详情页展示标题、图片材料、位置地图、类型、时间、用户、经纬度和辅助地址。
- 咨询/反馈详情页展示内容、图片和回复入口。
- 状态时间线展示每个办理节点、处理说明、操作者、时间和图片材料。
- 处理人员可在状态时间线中选择结束、继续跟进或解决，并上传处理材料。
- 用户可在解决后进入评价节点。

### 统计分析

- 首页看板展示本月新增、历史累计、已解决、无法处理、按时办结率、满意率等指标。
- 统计页使用 ECharts 展示：
  - 线索类型占比饼图。
  - 线索数量趋势柱状图。
  - 按日、月、年等视图补齐时间序列。
- 统计区间可选择起止日期。

| 文件 | 说明 |

```text

```

## 本地运行

### 安装依赖

```bash
npm install
```

### 配置接口地址

修改 `http/index.js`：

```js
export const baseURL = "http://localhost:8080"
```

如需使用独立资源域名，可同步调整 `config.js` 中的 `resourceURL`。

### 启动

使用 HBuilderX 打开项目，选择运行到浏览器、App 或小程序模拟器。运行前请确认后端 `poyang-report-backend` 已启动，并且移动端可访问后端地址。

## 典型使用流程

1. 用户进入“我的”页面登录。
2. 用户完成实名认证或查看已有认证状态。
3. 用户进入“举报”入口，选择举报、咨询或反馈。
4. 填写表单，必要时进行地图选点并上传现场图片。
5. 提交后在“我的上报”查看记录。
6. 处理人员进入“工作台”接收待办。
7. 在详情页和状态记录页更新办理节点、上传材料或回复。
8. 数据页查看整体线索分布和处理趋势。

## 配套后端

本项目依赖 `poyang-report-backend` 提供接口服务，默认端口为 `8080`。前后端主要接口路径包括 `/visitor`、`/user` 和 `/report`。

## 应用场景

项目适用于湖区生态巡查、城市治理、公众反馈、热线工单、基层事件处置和移动端办理协同等业务场景。
